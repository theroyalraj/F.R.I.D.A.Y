Privacy Policy

Your voice assistant for controlling this PC. Here is exactly what we do — and do not — do with your data.

Effective April 2026
🔒

No data leaves your machine by default

Friday is a self-hosted skill. Commands go from your Alexa device to your own server on your own PC — not to any third-party cloud service we operate. You own and control everything.

📥What information is processed

When you speak a command, the following passes through the Friday skill pipeline:

  • Voice command text — the NLU transcript Amazon produces (e.g. "open Notepad"). It is forwarded to your local N8N and pc-agent, used to run the task, then discarded.
  • Alexa userId — an opaque identifier Amazon assigns to your account. Used to match results back to you. Never shared externally.
  • Session metadata — requestId, locale, timestamp. Used for logging and debugging on your local machine only.
  • Task output — Claude's response is stored briefly in memory on your local skill-gateway (process restart clears it) so "last result" replies work.
Nothing is written to a remote database. All state is in RAM on your local server and disappears when the process restarts.

🚫What we never collect

  • Raw audio or voice recordings — Amazon handles speech-to-text; we only receive the text result.
  • Personal account details, email addresses, or payment information.
  • Browsing history, location, or data from other apps or skills.
  • Analytics, telemetry, or usage statistics of any kind.
  • Data from any other Alexa skill or Amazon service.

🔄How data flows

The full path for a spoken command:

  • Alexa cloud (Amazon) → transcribes your speech → sends an HTTPS request to your skill endpoint.
  • Skill-gateway (your PC) → verifies Amazon's signature → forwards the command text to your local N8N webhook.
  • N8N (your Docker container) → runs your workflow → calls your local pc-agent.
  • pc-agent / Claude (your PC) → executes the task locally → result stored in memory → reply spoken back via Alexa.
Every step after the Alexa cloud is entirely on your machine. Your commands never reach any server we operate.

🔔Proactive notifications (optional)

If you configure Alexa Proactive Events (to receive a chime when a task finishes), the following is sent to Amazon's Proactive Events API:

  • Your Alexa userId — to route the notification to your device only.
  • A short status label (e.g. "Friday — your task is done") — no command text, no output details.

This feature is off by default. You opt in by setting ALEXA_LWA_CLIENT_ID / ALEXA_LWA_CLIENT_SECRET in your local .env.

☁️Amazon's role

Alexa speech recognition and NLU are provided by Amazon. Amazon's own privacy policy governs how your voice data is handled before it reaches this skill. We receive only the text transcript Amazon provides.

See: Amazon Alexa Privacy Notice

🗓️Data retention

  • In-memory state (last result, awaiting-user prompt) — cleared when skill-gateway restarts. No persistence layer.
  • Logs — written to your local machine only if OPENCLAW_LOG_DIR is set. You control rotation and deletion.
  • N8N execution history — stored in your local Docker volume. Manage or delete from the N8N UI.

👤Children's privacy

Friday is a personal productivity tool intended for adults. It is not directed at children under 13 and does not knowingly process data from minors.

📝Changes to this policy

If this policy is updated, the effective date at the top of this page will change. Because this is a self-hosted skill you control, you will always have access to the latest version in your local repo at skill-gateway/public/privacy.html.

✉️Questions?

Friday is a self-hosted, personal-use skill. If you have questions about how data is handled, review the source code in your local Openclaw repo — it is all there.
